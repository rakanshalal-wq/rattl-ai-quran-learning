# User Flow

This document describes the public, non-sensitive product flow for Rattl.

## Parent Flow

1. Parent creates or accesses an account.
2. Parent manages children profiles.
3. Parent reviews memorization progress.
4. Parent follows activity and learning status from a simple dashboard.

## Child Flow

1. Child starts a guided memorization session.
2. The session presents a simple and focused interface.
3. The child listens, recites, and repeats.
4. Progress moves step by step.

## Learning Experience Goals

- Keep the interface calm.
- Avoid unnecessary distractions.
- Make the child session easy to understand.
- Give parents a clear summary of progress.
- Support Arabic-first learning.

## Privacy Note

This repository does not include real users, production data, private audio, backend routes, or AI evaluation logic.
